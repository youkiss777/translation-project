# 翻訳ワークフロー

## 1. 準備段階

- 原文ファイルを `source` ディレクトリに追加
- 翻訳対象のファイルを Issue で登録
- 担当者の割り当て

## 2. 翻訳段階

- 作業ファイルを `translations/in-progress` に作成
- 翻訳作業を進める
- コミットメッセージには進捗状況を含める

## 3. レビュー段階

- 翻訳完了後、Pull Request を作成
- レビュアーによるチェック
- フィードバックを反映

## 4. 完了段階

- レビュー承認後、`translations/completed` に移動
- Issue をクローズ
- 進捗報告の更新
